# Navigation

Menu e navegacao adaptavel por role.

---

## O Problema Universal

Sistemas com multiplos modulos precisam de navegacao clara. Usuarios precisam encontrar funcionalidades rapidamente. Menus confusos geram frustraÃ§Ã£o e subutilizacao do sistema.

---

## Perguntas de Descoberta

Antes de implementar, responda:

1. **Quais modulos existem no sistema?**
   - Core (operacao): inbox, calendar, metrics
   - Settings (configuracao): entity, users, integrations
   - User (pessoal): profile, notifications

2. **Como agrupar os modulos?**
   - Por funcao (operacao, configuracao, pessoal)?
   - Por frequencia de uso?
   - Por role?

3. **O que muda por role?**
   - Admin ve tudo?
   - Operador ve apenas operacao?
   - Gestor ve operacao + metricas?

4. **Qual a acao mais frequente?**
   - Deveria ter atalho/destaque?
   - FAB (floating action button)?

5. **Quais itens precisam de indicadores?**
   - Badge de pendentes (inbox: 5)?
   - Status (online/offline)?
   - Alertas?

6. **Mobile: qual o padrao?**
   - Bottom navigation?
   - Hamburger menu?
   - Tab bar?

## Fluxos

### Carregar Menu

```
1. Usuario faz login
2. Sistema busca roles do usuario
3. Para cada modulo:
   - Verificar se role tem acesso
   - Se sim, incluir no menu
4. Para modulos com badge:
   - Buscar contagem (inbox pendentes, notificacoes)
5. Renderizar menu filtrado
```

### Atualizar Badge em Tempo Real

```
1. Evento acontece (nova tarefa no inbox)
2. Backend emite evento via WebSocket/polling
3. Frontend atualiza badge sem refresh
4. Badge muda: (5) â†’ (6)
```

### Navegacao Mobile

```
1. Usuario esta em uma pagina
2. Toca em item do bottom nav
3. Navega para nova pagina
4. Item ativo destaca
5. Se pagina nao existe no bottom nav:
   - Manter ultimo item ativo
   - Ou mostrar indicador de "outro"
```

## Principios de Design

### 1. Hierarquia de Importancia

Modulos mais usados devem estar mais acessiveis:
- No topo da sidebar
- No bottom nav (mobile)
- Com icone reconhecivel

### 2. Consistencia de Posicao

Usuario aprende onde as coisas estao. Nao mude posicao entre paginas ou apos updates.

### 3. Feedback Visual

Item ativo deve ser claramente diferente. Usuario sabe onde esta.

### 4. Badges Atualizados

Badge desatualizado e pior que nenhum badge. Se mostrar (5), devem ser 5 mesmo.

### 5. Agrupamento Logico

Itens relacionados juntos. "Configuracoes" agrupadas separadas de "Operacao".

### 6. Mobile e Diferente

Nao tente encaixar sidebar de desktop em mobile. Use bottom nav ou drawer. Priorize os 4-5 itens mais importantes.

---

## Anti-patterns

### "Tudo em um nivel"
**Problema:** 20 itens de menu sem agrupamento.
**Solucao:** Agrupar por categoria (Operacao, Configuracoes, etc).

### "Menu muda por pagina"
**Problema:** Usuario perde referencia de onde esta.
**Solucao:** Menu consistente em todas as paginas.

### "Badge estatico"
**Problema:** Mostra (5) mas tem 10 pendentes.
**Solucao:** Atualizar em tempo real ou no carregamento.

### "Mobile = sidebar escondida"
**Problema:** Usuario precisa de 3 cliques para navegar.
**Solucao:** Bottom nav para acesso rapido aos principais.

### "Icones sem label"
**Problema:** Usuario nao sabe o que cada icone significa.
**Solucao:** Label visivel ou tooltip. Ao menos em desktop.

### "Muitos niveis de submenu"
**Problema:** Configuracoes > Sistema > Geral > Opcoes...
**Solucao:** Maximo 2 niveis. Se precisar mais, repensar estrutura.
